# UnLha64x コマンドラインスイッチ対応ドキュメント

UnLha64x (`Unlha()` / `UnlhaW()`) が解釈・処理するコマンドラインスイッチの対応状況およびその詳細動作について記述します。

---

## 1. 対応しているスイッチ

指定された際に DLL 内部および LhaCore の動作に反映されるスイッチです。

| スイッチ | 説明 | 動作詳細 |
| :--- | :--- | :--- |
| `--`| 指定文字の変更 | レスポンス指定文字およびスイッチ文字を変更します (詳細は後述) |
| `-d` <br> `-d1` | ディレクトリー格納 / 展開 | **展開時**: ディレクトリ構造を維持して展開します (`ignore_directory = FALSE`)。<br>**圧縮時**: サブディレクトリを再帰的に検索して圧縮します (`recursive_archiving = TRUE`)。 |
| `-d0` | ディレクトリー構造の無視 | **展開時**: ディレクトリ構造を無視し、指定フォルダ直下にすべてのファイルを平坦に展開します (`ignore_directory = TRUE`)。<br>**圧縮時**: サブディレクトリの再帰検索を行いません (`recursive_archiving = FALSE`)。 |
| `-r` <br> `-r1` <br> `-r2` | 再帰的検索の有効化 | サブディレクトリを再帰的に検索して圧縮対象にします (`recursive_archiving = TRUE`)。 |
| `-r0` | 再帰的検索の無効化 | サブディレクトリの再帰検索を行いません (`recursive_archiving = FALSE`)。 |
| `-x` <br> `-x1` | ディレクトリー名を有効にする | **展開時**: ディレクトリ構造を維持して展開します (`ignore_directory = FALSE`)。<br>**圧縮時**: 書庫内にパス情報を記録します (`generic_format = FALSE`)。 |
| `-x0` | ディレクトリー名を除外する | **展開時**: ディレクトリ構造を無視して展開します (`ignore_directory = TRUE`)。<br>**圧縮時**: 書庫内にパス情報を格納せず、ファイル名のみで平坦に格納します (`generic_format = TRUE`)。 |
| `-jm`<br>(`-jm0`〜`-jm8`) | 圧縮メソッド（形式）指定 | 指定された値（0〜8）に応じて、LHAの圧縮アルゴリズム（メソッド）を切り替えます。<br>- `-jm0` : `-lh0-` (無圧縮)<br>- `-jm1` : `-lh1-`<br>- `-jm2` : `-lh5-` (デフォルト)<br>- `-jm3` : `-lh6-`<br>- `-jm4` : `-lh7-`<br>- `-jm5` : `-lh2-`<br>- `-jm6` : `-lh3-`<br>- `-jm7` : `-lzs-`<br>- `-jm8` : `-lz5-` |
| `-jx<pattern>` | 除外ファイルパターンの指定 | 指定されたパターンに一致するファイルを圧縮対象から除外します。内部的に Unix LHa 互換の `-x<pattern>` 引数へと変換されて LhaCore に引き渡されます。 |
| `-gl<filename>` | ログファイルの出力 | 処理終了後、指定された `<filename>` パスに対して実行ログ（進捗および結果メッセージ）を出力します。 |
| `-y` | 応答の強制「はい」化 | 展開等の確認プロンプトにおいて、応答をすべて「はい」として扱い、上書き確認なしで強制実行します (`force = TRUE`)。<br>また、上書き対象の既存ファイルが「読み取り専用（ReadOnly）」属性である場合、事前に属性を自動解除して上書きを実行する処理が有効になります。 |

## 2. 無視しているスイッチ

エラーにはせず、引数解析時にリストから取り除かれて無視されるスイッチです（現状の動作に影響を与えません）。

| スイッチ | 説明 / 動作状況 |
| :--- | :--- |
| `-+` | レジストリ設定に関するスイッチ。設定をレジストリから読み書きしないため、単純に無視されます。 |
| `-ga`<br>(`-ga0`, `-ga1` など) | 展開時の読み取り専用（ReadOnly）ファイルの属性解除指定。<br>本DLLでは `-ga` の指定有無にかかわらず、**常に読み取り専用ファイルを自動で解除して上書きする仕様**となっているため、このスイッチは安全に無視されます。 |
| `-ga`, `-gn`, `-gp`, <br> `-jf`, `-jyo`, `-m`, <br> `-c`, `-n` 等の各数値付き | ツール連携や特定の動作制御用。非サポートですが、エラーにせずそのまま無視されます。 |
| `a`, `c`, `e`, `f`, `h`, `i`, <br> `l`, `m`, `n`, `p`, `s`, `t`, <br> `u`, `w`, `z` など | コマンドオプションとしてのスイッチ指定。無視されます。 |
| `ja` 〜 `jz` の各2文字 | `j` で始まる2文字のスイッチ群（`-jf`, `-jyo` などを除く `ja`〜`jz`）。無視されます。 |
| `gb`, `gd`, `ge`, `gf`, `gq`, <br> `gr`, `gt`, `gu`, `gx`, `gy` | `g` で始まる2文字のスイッチ群。無視されます。 |
| `k`, `v` | その他互換性のために無視されるスイッチです。 |

---

## 3. エラーとなる（非対応）スイッチ

以下のスイッチが指定された場合は非対応とみなし、即座にエラーコード `87` (`ERROR_INVALID_PARAMETER`) を返して処理を中断します。

* **`-j`**（スイッチの意味の交換）
* **`-jr`**（ファイルの置き換え）
* **`-jw`**（自己解凍書庫の作成）
* **`-gw`**（自己解凍書庫の作成）
* **`-gj`**（x 命令を e 命令とみなす）

## 4. 引数解釈と拡張機能

UnLha64x では、オリジナルの UNLHA32.DLL との互換性向上のため、以下のような引数解析ルールおよび拡張機能を実装しています。

### A. スイッチの連続指定（まとめ指定）の自動分解
`-d1n1` のように、複数のスイッチが1つの引数として連結して指定された場合、内部で自動的に個別の引数に分解（例：`-d1` と `-n1`）して解釈します。

* **分解ルール**:
  * `j` または `g` で始まる2文字スイッチ（`jx`, `gl`, `gb` など）と、それ以外の1文字スイッチを正しく判別して分解します。
  * **文字列引数を取るスイッチ**: スイッチ引数として任意の文字列を取る以下のスイッチが検出された場合、それ以降の引数文字列全体をそのスイッチの値とみなして分解を終了します。
    * `w`, `z`, `jx`, `gl`, `gb`, `gr`, `jz`, `jo`, `jtz`, `gyd`, `gye`, `gyt`, `gyw`
  * **その他のスイッチ**: 次にアルファベット（新たなスイッチの開始）が現れるまでの数字や記号（`0`, `1`, `2`, `+`, `-` など）をそのスイッチの値とし、アルファベットが現れた位置から次のスイッチとして分解します。

### B. レスポンスファイル対応
コマンドラインの引数として `@{ファイルパス}`（例: `@C:\Temp\response.txt`）が指定された場合、それをレスポンスファイルとして扱い、ファイルに記述された内容を展開して引数として処理します。

* **書式**: コマンドラインと同様に、空白、タブ、改行（`\r`, `\n`）を引数の区切り文字として認識します。
* **文字コードの自動判定**: レスポンスファイルのエンコーディングは自動的に判定されます。
  * `UTF-16LE` (BOM `FF FE` あり)
  * `UTF-8` (BOM `EF BB BF` あり)
  * BOMなしの場合は、まず `UTF-8` としてデコードを試み、成功すれば UTF-8 として扱い、失敗した場合は `ANSI (CP932)` として読み込みます。
* **制限事項**: レスポンスファイル内でさらに別のレスポンスファイルを指定するネスト（多重展開）はサポートしていません。

### C. `--` オプションによるレスポンスファイル動作の制御
レスポンスファイル機能の有効化・無効化や、レスポンスファイルを指定する際の開始文字（デフォルトは `@`）を変更するスイッチです。

* **`--`** または **`--1`**:
  * レスポンスファイル指定文字（`@`）による展開機能を無効化します。引数内の `@` で始まるパラメータは、通常のファイル名などの引数として扱われます。
* **`--0`**:
  * レスポンスファイル機能およびスイッチ文字を初期状態（レスポンス文字: `@`、スイッチ文字: `-` / `/`）にリセットします。
* **`--2`**:
  * レスポンスファイル機能とスイッチ文字の双方を無効化します。
* **`--3`**:
  * スイッチ文字（`-` / `/`）のみを無効化します。
* **`--<文字>`** (例: `--!`):
  * レスポンスファイルを指定するためのプレフィックス文字を変更します。例えば `--!` と指定した場合、`!C:\Temp\response.txt` のように `!` から始まる引数をレスポンスファイルとして扱うようになります。
